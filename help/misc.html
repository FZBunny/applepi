Miscellaneous


Running ApplePi

During installation, a copy of the ApplePi desktop file should have been dropped on your desktop. If not, you may manually drag it to your desktop from:
/usr/share/applepi/applepi.desktop
You may also run ApplePi from a terminal with:  applepi
Running from a terminal is necessary when using most of the Debug tools.   When debugging, it is sometimes helpful to save terminal output to a file for later inspection like this:
applepi | tee somefilename
Debug output will still be visible on the terminal.

Booting from disk

When ApplePi is turned on, it (i.e. the internal ROM) will search for a disk controller containing a disk in decreasing slot order beginning with slot 7 (the "HD 1" button), and boot from the first bootable image it finds.
A boot may be interrupted by pressing the 'RESET' button, and restarted with  PR#6  or   PR#7

Disks

ApplePi uses disk images of three types:
- In "Floppy 1 & 2" drives: 5¼" disk images in 140K (143360 byte) files.
- "HD 1 & 2" drives: 3½" disk images in 800K (819200 byte) files, or hard drive images in up to 32MB files.

Below each disk button is a lable identfying the loaded disk:

- DOS3 disks show "DOS 3 Vol.", followed by a volume number, 0-255. For various obscure reasons, this is almost aways "254".
Example:    DOS3 Vol.254

- ProDOS8 disks show a volume name, preceded by a "/".   Note that the volume name is not  the name of the file containing the disk image.
Example:    /APPLEWORKS

- Apple Pascal disks show the Pascal volume name, which is always terminated by a colon.   Again, this is unrelated to the name of the image file.
Example:    APPLE1:
