Printing


Printing text is accomplished by use of the PR#1 command.
Printing a PDF is usually done with programs such as 'Print Shop'.

Case 1: The 'Print to File' menu is not in effect:

By default, text output will be directed to the terminal from which ApplePi has been executed.  (If ApplePi was not executed from a terminal, text is simply sent into the aether, gone forever.)

Case 2: 'Print Text' was used:

Text will be written to the selected file.  Every byte will be flushed to disk as it is encountered, so the 'Close Printer' button may be largely ignored unless you wish to close that file, and open another.  If you exit ApplePi without closing the file, no text will be lost.

This is very unlike printing a PDF:

Case 3: 'Print a PDF' was used:

Data representing dot-matrix graphics will be accumulated in a buffer inside ApplePi.  It will not be written to the PDF file until the 'Close Printer' button is clicked.    If the 'Close Printer' button is not clicked before ApplePi is shut down, the PDF contents will be lost.

A few tricks

If, for some reason, you don't wish to use PR#1, there are still some ways to get text output to a file:

1)  In a terminal window:

* Enter:     applepi > somefilename.txt
Then select the 'Preferences->Echo Screen Text to Terminal' menu item.
* Or use:    applepi >> somefilename.txt     to append text to an existing file.
* If you wish to see the text on the terminal as it is written to the file, use:
applepi | tee somefilename.txt

2)  Select the 'Preferences->Echo Screen Text to File...' menu item.

Any of these 'trick' methods will echo any text which appears the screen to a disk file, including all entered commands, whether in Basic or in the Apple II monitor.
