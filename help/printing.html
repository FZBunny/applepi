Printing


Printing is accomplished by use of the PR#1 command.

Case 1: The 'Print to File' menu is not in effect:

By default, text output will be directed from the screen to the terminal from which ApplePi has been executed.  (If ApplePi was not executed from a terminal, text is simply sent into the aether, gone forever.)

Case 2: 'Print Text' was used:

Text will be written to the selected file.  Every byte will be flushed to disk as it is encountered, so the 'Close Printer' button may be largely ignored unless you wish to close that file, and open another.  If you exit ApplePi without closing the file, no text will be lost.

This is very unlike printing a PDF:

Case 3: 'Print a PDF' was used:

Data representing dot-matrix graphics will be accumulated inside ApplePi.  It will not be written to the PDF file until the 'Close Printer' button is clicked.
